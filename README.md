# 🥭 Green Mango Marketplace - Senior MVP

A high-performance guest posting marketplace built with the **Next.js 16 (App Router)**, **Prisma**, **Neon PostgreSQL**, and **NextAuth.js**. This platform features role-based access control (RBAC), secure cryptographic payment verification, and context-aware identity management.

---

## 🚀 Architectural Highlights

### 1. Context-Aware Identity Management (Smart OAuth)
The platform utilizes **NextAuth.js v4** integrated with **Google Cloud Console**. Unlike standard implementations, Green Mango captures the user's intent *before* the OAuth dance. 
- **Smart Redirects**: Triggering login from the "Seller" landing page passes a contextual role via `callbackUrl`.
- **Automatic Sync**: A dedicated `DashboardSetup` handler ensures new users are created in the Neon DB with the correctly mapped role without requiring manual profile setup forms.

### 2. RBAC & Security Middleware
A centralized **Next.js Middleware** enforces Role-Based Access Control at the edge.
- **Token-Level RBAC**: Roles are baked into the **JWT session**, preventing unauthorized access to the Admin and Seller panels even if the URL is known.
- **Session Stability**: Integration between `getServerSession` and Prisma ensures that all API requests are verified against real-time database state.

### 3. Cryptographic Payment Handshake
The guest post booking flow uses a secure two-phase handshake with **Razorpay**.
- **Atomic Transactions**: Orders are only persisted to the database inside a `prisma.$transaction` after the payment signature is verified using the `crypto` library.
- **Price Integrity**: Razorpay Order IDs are generated by fetching the current price from the server-side database, preventing client-side price manipulation.

### 4. Senior UI/UX Standards
- **Framer Motion**: Smooth micro-interactions and layout transitions across all dashboards.
- **Zero-Scroll Auth**: Login and Register pages are optimized for viewports, ensuring a "native-app" feel.
- **High Contrast Design**: Rigorous contrast audit ensuring Web Content Accessibility Guidelines (WCAG) compliance for professional clarity.

## 🛠️ Technical Stack
- **Framework**: Next.js 16 (App Router)
- **Database**: Neon (Serverless PostgreSQL)
- **ORM**: Prisma
- **Auth**: NextAuth.js (Google OAuth + Credentials)
- **Payments**: Razorpay Node SDK
- **Styling**: Tailwind CSS + Framer Motion

## 🧪 Quick Start & Verification

### Running Locally
1. Clone the repository.
2. Install dependencies: `npm install`.
3. Configure `.env` with the provided Google and Razorpay credentials.
4. Push the schema: `npx prisma db push`.
5. Seed the database: `npm run dev` and visit `/api/seed`.

### Verification Checklist for Reviewer
- [X] **Admin Login**: Test with `admin@test.com`.
- [X] **Google OAuth**: Test "Continue with Google" as both Buyer and Seller.
- [X] **Razorpay**: Place a test order on the Marketplace using the "Success" demo card.

---
